## 設計手順

1. **食体験を定義する**: 食べ手、価格帯、季節、食事場面、量、持ち運び時間、酒との関係、設備、仕込み人数、提供速度を整理する。不明点は妥当な仮定を明示する。
2. **一文のコンセプトを置く**: 「春の苦味と炭火香を、軽い酸味でつなぐ晩酌」など、季節・主役・感情・技法が伝わる一文にする。
3. **主役と脇役を決める**: 主役食材は原則1つ。副菜や薬味は主役を強める役割を与える。意味のない高級食材、飾り、異文化要素を足さない。
4. **味を設計する**: 塩味・甘味・酸味・苦味・うま味に、辛味、脂肪、香り、温度を加えて強弱を記述する。濃い味を重ねず、酸味・苦味・香味・淡味で休符を作る。
5. **食感と調理法を設計する**: やわらかい／歯切れ、しっとり／乾いた、滑らか／粒、温／冷を対比させる。同じ加熱法や同じ食感を献立内で反復しすぎない。
6. **色と形を設計する**: 赤・黄・緑（青）・白・黒／褐色を確認し、主色・補助色・差し色を決める。高さ、余白、方向、器・容器とのコントラスト、食べる順序を設計する。
7. **文化の軸を守って創造する**: まず伝統の味型、基礎技法、食べ方を特定する。その核を1つ以上残し、変える要素は1～2点に絞る。「調味料を替えただけ」の安易な融合を避ける。
8. **現場へ落とす**: 歩留まり、再利用可能な仕込み、同時調理、保持時間、温度変化、盛付け時間、廃棄、原価を確認する。
9. **試作する**: グラム、温度、時間、鍋・火力、出来高を記録する。一度に変える変数は1つ。調理直後だけでなく、想定提供時点でも官能評価する。
10. **採点して修正する**: 下記基準を各5点で採点し、3点未満を修正する。

## 必須評価

| 評価軸 | 合格条件 |
|---|---|
| コンセプト | 一文で差別化と食べる理由を説明できる |
| おいしさ | 味・香り・温度・食感に山と休符がある |
| 文化的整合 | 料理の核となる技法・味型・食べ方が説明できる |
| 季節性 | 旬、気候、器、香りのいずれかが必然として働く |
| 視覚 | 主役が明確で色・形・余白に意図がある |
| 実現性 | 設備、人数、時間、原価で再現可能 |
| 提供品質 | 食べる時点で狙った味・食感を保つ |
| 安全性 | 加熱、冷却、交差汚染、アレルゲンを確認済み |
| 独自性 | 定番との差を食材名以外で説明できる |

## 出力形式

提案時は最低限、次を示す。

1. コンセプトと狙う食体験
2. 献立または料理名と役割
3. 味・香り・食感・色・温度の設計
4. 文化的な核と創造した点
5. レシピ骨格（重量比、重要温度・時間、仕上げ順）
6. 仕込み・提供・弁当なら時間経過への対策
7. 食品安全・アレルゲン上の注意
8. 代替案と試作で検証すべき仮説

既視感のある案しか出ない場合、食材を奇抜にする前に「季節の解像度」「香りの層」「食感差」「技法」「食べる動作」「物語」のいずれかを深掘りする。

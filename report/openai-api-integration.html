GPT-5.5 弁当提案API
実装根拠

確認日：2026-07-13。OpenAI公式ドキュメントのみを利用した。

1. GPT-5.5 Model

発行元・種別
OpenAI Developers／公式モデル仕様
URL
https://developers.openai.com/api/docs/models/gpt-5.5
確認事項
正式なモデルIDはgpt-5.5。Responses API、Structured Outputs、reasoning effortをサポートする。
実装への反映
サーバー側のResponses APIでモデルをgpt-5.5、推論強度をhighに固定。
信頼度・限界
非常に高い。モデル利用可否とレート制限はOpenAIアカウントの利用階層に依存する。

2. Structured model outputs

発行元・種別
OpenAI Developers／公式APIガイド
URL
https://developers.openai.com/api/docs/guides/structured-outputs
確認事項
スキーマに準拠した構造化レスポンスを生成できる。
実装への反映
ZodスキーマとzodTextFormatを使用し、候補数を4件、材料・手順・色・安全情報を必須化。
信頼度・限界
非常に高い。料理内容の品質はプロンプトとモデル出力の試食・人間レビューを別途必要とする。

3. Responses API migration guide

発行元・種別
OpenAI Developers／公式APIガイド
URL
https://developers.openai.com/api/docs/guides/migrate-to-responses
確認事項
Responses APIを統合された生成エンドポイントとして利用する構成。
実装への反映
Next.js Route HandlerからのみOpenAIへ接続し、APIキーをクライアントへ公開しない。
信頼度・限界
非常に高い。VercelにはOPENAI_API_KEYの環境変数設定が必要。

実装判断

ユーザー指定を優先してgpt-5.5を採用した。公式モデル一覧ではGPT-5.6も掲載されているが、本作業では自動的にモデルを変更しない。料理人ペルソナと配分規則は.agents/skills/culinary-menu-foundation/references/bento-ai-personas.mdへ保存し、同等の制約をサーバープロンプトへ反映した。
